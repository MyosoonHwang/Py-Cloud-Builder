# ☁️ NHN Cloud Network Auto-Builder

**NHN Cloud (OpenStack)** 환경에서 VPC와 서브넷을 쉽고 안전하게 관리하는 파이썬 자동화 도구입니다.
복잡한 콘솔 작업 없이, 터미널에서 **리소스를 시각적으로 조회**하고 **중복 걱정 없이 생성**할 수 있습니다.

## ✨ 주요 기능 (Key Features)

* **🔐 하이브리드 인증 (Smart Auth):**
    * 아이디/비밀번호만 입력하면 **참여 중인 프로젝트(Tenant)를 자동으로 검색**해 연결합니다.
    * 자동 조회가 실패할 경우, Tenant ID 수동 입력 모드로 자동 전환되어 끊김 없이 작업할 수 있습니다.
* **🛡️ 3중 스마트 유효성 검사:**
    1. **CIDR 중복 방지:** 기존 VPC 대역과 겹치는지 계산하여 경고하고, 사용 가능한 대역을 추천합니다.
    2. **이름(Name) 중복 방지:** 이미 존재하는 VPC나 서브넷 이름을 사용할 수 없도록 막습니다.
    3. **범위(Scope) 검사:** 서브넷이 VPC의 CIDR 범위를 벗어나지 않는지 검증합니다.
* **👀 스마트 필터링 (Smart Filtering):**
    * NHN Cloud의 공용 네트워크(`Public Network`)나 다른 프로젝트의 리소스는 제외합니다.
    * 오직 **내가 현재 프로젝트에서 생성한 리소스**만 깔끔하게 필터링하여 보여줍니다.
* **📊 트리 뷰 시각화:** VPC와 그 안에 속한 서브넷 구조를 계층형 트리(Tree) 형태로 직관적으로 출력합니다.
* **⚡ .env 환경 변수 지원:** 매번 로그인할 필요 없이 `.env` 파일에 계정 정보를 저장하여 **원터치 로그인**이 가능합니다.

## 🛠️ 설치 및 설정 (Setup)

### 1. 필수 라이브러리 설치
이 도구는 `requests`와 `python-dotenv` 라이브러리를 사용합니다.

```bash
pip install requests python-dotenv
```

2. .env 파일 설정 (선택 사항)
프로젝트 루트 경로에 .env 파일을 생성하고 아래 내용을 작성하면, 실행 시 자동으로 정보를 불러옵니다. (보안을 위해 이 파일은 GitHub에 업로드하지 마세요!)

코드 스니펫
NHN_ID=본인아이디
NHN_PW=본인API비밀번호
# NHN_TENANT_ID=특정_테넌트_ID (선택사항: 고정하고 싶을 때만 입력)
⚠️ 중요: API 비밀번호 NHN_PW에는 웹사이트 로그인 비밀번호가 아닌 API 전용 비밀번호를 입력해야 할 수 있습니다.

확인 경로: NHN Cloud 콘솔 > 우측 상단 회원 아이콘 > API 보안 설정 > API 비밀번호 설정

🚀 사용 방법 (Usage)
프로그램을 실행하면 대화형 메뉴가 나타납니다.
```Bash
python main.py
```
메인 메뉴
📊 조회 (List): 현재 프로젝트 내의 VPC 및 서브넷 현황을 트리 구조로 확인합니다.

🏗️ 생성 (Create): 안내에 따라 VPC와 서브넷을 생성합니다. (이름/CIDR 중복 검사 포함)

❌ 종료: 프로그램을 종료합니다.

❗ 트러블슈팅 (FAQ)
Q. 401 Client Error 또는 인증 실패가 뜹니다.

A. 아이디는 맞지만 비밀번호가 틀린 경우입니다. 웹사이트 비밀번호 대신 API 비밀번호를 설정하여 다시 시도하세요.

Q. 자동 조회가 실패하고 Tenant ID를 입력하라고 합니다.

A. API 권한 문제로 프로젝트 목록 조회가 막힌 경우입니다. 당황하지 말고 NHN Cloud 콘솔에서 **Tenant ID(Project ID)**를 복사해 입력하면 정상적으로 진행됩니다.
